- **목표**
  - 프론트 화면과 컴포넌트에 직접 박힌 Tailwind 글자 크기 유틸리티를 `frontend/src/main.css`에서 한 번에 관리할 수 있도록 정리한다.
  - `text-xs`, `text-sm`, `text-base`, `text-lg`, `text-xl`, `text-2xl`, `text-3xl`, `text-5xl`, `text-[10px]`, `text-[11px]` 사용을 전역 타입 스케일 클래스로 치환한다.

- **범위**
  - 글자 크기 변경과 직접 관련된 Tailwind 유틸리티만 대상으로 한다.
  - 레이아웃 유틸리티(`flex`, `grid`, `gap-*`, `px-*`, `rounded-*` 등)는 이번 범위에서 제외한다.
  - 색상 유틸리티 중 프로젝트 공통 클래스(`app-text-*`, `app-bg-*`)는 유지한다.

- **전역 관리 방식**
  - `frontend/src/main.css`의 `:root`에 타입 스케일 변수를 추가한다.
  - `frontend/src/main.css`의 `@layer utilities`에 `app-type-*` 클래스를 추가한다.
  - 각 화면 파일은 Tailwind `text-*` 크기 유틸리티 대신 `app-type-*` 클래스를 사용한다.

- **전역 타입 스케일**
  - `app-type-2xs` — 기존 `text-[10px]` 또는 작은 보조 라벨
  - `app-type-11` — 기존 `text-[11px]`
  - `app-type-xs` — 기존 `text-xs`
  - `app-type-sm` — 기존 `text-sm`
  - `app-type-base` — 기존 `text-base`
  - `app-type-lg` — 기존 `text-lg`
  - `app-type-xl` — 기존 `text-xl`
  - `app-type-2xl` — 기존 `text-2xl`
  - `app-type-3xl` — 기존 `text-3xl`
  - `app-type-4xl` — 예비 확장용
  - `app-type-5xl` — 기존 `text-5xl`

- **리팩토링 정책**
  - 화면 템플릿의 `text-xs`는 `app-type-xs`로 치환한다.
  - 화면 템플릿의 `text-sm`은 `app-type-sm`으로 치환한다.
  - 화면 템플릿의 `text-base`는 `app-type-base`로 치환한다.
  - 화면 템플릿의 `text-lg`는 `app-type-lg`로 치환한다.
  - 화면 템플릿의 `text-xl`은 `app-type-xl`로 치환한다.
  - 화면 템플릿의 `text-2xl`은 `app-type-2xl`로 치환한다.
  - 화면 템플릿의 `text-3xl`은 `app-type-3xl`로 치환한다.
  - 화면 템플릿의 `text-5xl`은 `app-type-5xl`로 치환한다.
  - 화면 템플릿의 `text-[10px]`는 `app-type-2xs`로 치환한다.
  - 화면 템플릿의 `text-[11px]`는 `app-type-11`로 치환한다.
  - `FactoryLineMasterView.vue`의 scoped CSS `@apply text-*`는 `font-size: var(--app-font-size-*)`와 `line-height: var(--app-line-height-*)`로 치환한다.
  - `main.css` 내부 공통 컴포넌트의 `@apply text-*`도 동일하게 타입 스케일 변수 기반으로 치환한다.

- **검증 기준**
  - `frontend/src`에서 직접 글자 크기 Tailwind 유틸리티 검색 결과가 없어야 한다.
  - `npm run build`가 통과해야 한다.
