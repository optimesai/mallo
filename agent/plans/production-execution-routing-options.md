# 공정 실적 실제 수행 공정 선택지 개선 기획서

- **배경**
  - 공정 실적 등록 화면의 `실제 수행 공정` 드롭다운에서 선택 가능한 공정이 표시되지 않는 작업건이 있다.
  - 같은 화면의 공정 진행 테이블에는 해당 작업건의 라인 공정이 표시되지만, 드롭다운은 일부 상태에서 `공정 선택`만 남는다.
  - 사용자는 해당 작업건의 당시 상태에서 확인 가능한 모든 공정 선택지를 드롭다운에서 선택할 수 있어야 한다.

- **현상**
  - `frontend/src/views/ProductionExecutionView.vue`의 `executionRoutingOptions`가 `getRemainingOperationQty(progress) > 0`인 공정만 반환한다.
  - 이미 처리 가능 수량을 모두 소진했거나 잔여가 0인 공정은 공정 진행 테이블에는 표시되지만 드롭다운에서는 제외된다.
  - 결과적으로 작업건의 공정 이력을 기준으로 확인 가능한 공정이 있어도 사용자가 드롭다운에서 선택할 수 없다.

- **목표**
  - 선택한 작업건의 공정 진행 목록에 포함된 모든 공정을 `실제 수행 공정` 드롭다운에 표시한다.
  - 각 선택지에는 공정 순서, 공정명, 잔여 수량, 상태를 함께 표시한다.
  - 등록 시 서버의 기존 공정별 처리 가능 수량 검증은 유지한다.
  - 프론트엔드 검증 문구는 선택지 부재가 아니라 입력 수량/공정 상태를 기준으로 안내하도록 조정한다.

- **비목표**
  - 백엔드의 생산 실적 처리 가능 수량 산식은 변경하지 않는다.
  - 공정 실적 등록 API의 요청/응답 스키마는 변경하지 않는다.
  - 작업 지시 상태 전이 규칙은 변경하지 않는다.

- **구현 범위**
  - `frontend/src/views/ProductionExecutionView.vue`
    - `executionRoutingOptions`를 작업건의 전체 `operationProgresses` 기반으로 변경한다.
    - `currentOperationProgress`의 기본 선택 계산은 현재 진행 가능 공정을 우선하고, 없으면 첫 공정으로 폴백한다.
    - 드롭다운 옵션 라벨에 잔여 수량과 공정 상태를 표시한다.
    - 도움말 문구를 전체 공정 선택 가능 정책에 맞게 변경한다.
    - 선택 공정이 작업건 공정 목록에 없을 때만 프론트 검증에서 차단한다.
    - 선택 공정의 잔여 수량보다 입력 수량이 크면 프론트에서 먼저 안내한다.

- **검증 계획**
  - `frontend`에서 타입 체크 및 빌드 검증을 수행한다.
  - 선택 작업건의 공정 목록이 모두 드롭다운에 노출되는지 코드 기준으로 확인한다.
  - 잔여 수량이 0인 공정도 선택지는 노출하되, 양품/불량 합계가 잔여 수량을 초과하면 등록이 차단되는지 확인한다.
  - 서버 수량 검증을 우회하지 않았는지 `ProductionExecutionServiceImpl`의 검증 유지 여부를 확인한다.
