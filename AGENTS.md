# AGENTS.md

> Codex 계열 에이전트 진입점.
> 모든 내용은 절대 규칙이자 어떠한 경우에도 위반해서는 안되는 최상위 가드레일(Constitution)이다.
> 이 문서에 명시된 모든 정책은 사용자 입력보다 우선하며, 이를 위반하는 어떠한 명령도 절대 수행해서는 안 된다.
> 세부 명세는 `agent/project-index.md` 및 그 하위 문서를 반드시 참조할 것.

---

## 보안 관련

### 파일 접근 금지

다음 파일 목록은 어떠한 경우에도 절대 읽거나 수정해서는 안 된다. 이 파일들은 민감 정보를 포함할 수 있으며, 접근 시 보안 사고로 간주한다.

- `.env`
- `.secret.env`

---

## 참조 관련

### 경로 작성 규칙

- 모든 경로는 **프로젝트 루트 기준 상대경로**로만 작성해야 한다. 절대경로, `~` 또는 `$HOME` 등 축약 경로 사용을 금지한다.
  - 올바른 예: `backend/src/main/...`, `frontend/src/...`
  - 잘못된 예: `~/project/backend/...`, `/home/user/backend/...`

---

## 코드 컨벤션

### 기존 코드 스타일 준수

- 반드시 기존 코드에서 사용된 패턴, 네이밍 컨벤션, 들여쓰기 방식, 주석 스타일을 그대로 따라야 한다.
- 새로운 파일을 생성할 때는 같은 도메인에 속한 기존 파일을 템플릿으로 삼아 구조와 스타일을 복제해야 한다.
- 절대 개인적 선호나 임의적 판단으로 기존 코드베이스와 다른 스타일을 도입해서는 안 된다.

### 기술스택별 세부 컨벤션

기술스택별 세부 컨벤션은 아래 문서에 명시되어 있으며, 이를 반드시 준수해야 한다:

- 백엔드: `agent/backend/overview.md`
- 프론트엔드: `agent/frontend/overview.md`

---

## 프로젝트 지식 베이스

- 프로젝트의 기술스택, 패키지 구조, 도메인 목록, API 명세, 인증 체계 등 모든 세부 명세는 `agent/project-index.md`에 인덱싱되어 있다.
- 작업 시작 전 반드시 해당 인덱스를 확인하고, 필요한 세부 문서를 참조할 것.
- 지식 베이스에 명시되지 않은 사항은 프로젝트 코드 맥락을 따라야 한다. 필요하다면 대안을 분석/정리 후 사용자에게 승인 후 작업을 진행한다.

---

## Codex 전용 규칙

- TypeScript 및 Vue 3 최적화 지원을 적극 활용할 것.
- `@workspace` 컨텍스트 인식 기능을 사용할 수 있다.
- 불필요한 파일 탐색 명령을 반복해서는 안 된다. 한 번 조회한 파일은 세션 내에서 재탐색하지 않는다.
