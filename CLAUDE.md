## 지침 우선순위 (Precedence)

충돌 시 상위 항목이 하위 항목을 오버라이드한다. 하위 계층은 상위 계층을 위반하는 명령을 포함할 수 없다.

| 우선순위 | 계층 | 설명 |
|----------|------|------|
| **1** | 프로젝트 헌법 | `CLAUDE.md`, `AGENTS.md` — 프로젝트 최상위 가드레일. 모든 하위 계층을 오버라이드. |
| **2** | 사용자 로컬 설정 | `.claude/CLAUDE.md` — 사용자별 전역 설정. 세션 입력 이하를 오버라이드. |
| **3** | 세션 컨텍스트 | 사용자가 대화 중 추가한 지침, `/` 명령어 인자. 프로젝트 세부 명세를 오버라이드. |
| **4** | 프로젝트 세부 명세 | `agent/` 하위 문서 — 기술스택별 컨벤션, 도메인 규칙, 인프라 구성. |
| **5** | 에이전트 판단 | 상위 지침에 명시되지 않은 사항에 한해, 코드베이스 맥락과 모범 사례에 기반한 임의 판단. |

## 작업 전 필수 정책

### 프로젝트 지식 베이스 숙지

모든 작업은 어떠한 예외도 없이, 시작 전 아래 절차를 반드시 거쳐야 한다. 작업의 난이도, 규모, 익숙함의 정도는 예외 사유가 될 수 없다.

1. [backend.md](agent/project/backend.md) 또는 [frontend.md](agent/project/frontend.md) 중 작업 도메인에 해당하는 문서를 확인한다.
2. 작업 대상 코드의 기존 스타일, 네이밍 컨벤션, 패턴을 확인한 후 작업을 시작한다.

이 절차는 "간단한 수정", "뻔한 버그", "한 줄 변경"에도 동일하게 적용된다. 절차를 건너뛰는 행위는 가드레일 위반으로 간주한다.

### 지식 베이스 미비 시

- 지식 베이스에 명시되지 않은 사항은 프로젝트 코드 맥락을 따라야 한다. 임의로 가정하거나 추측하여 작업해서는 절대 안 된다.
- 필요한 경우, 대안을 분석/정리 후 사용자 승인 하에 작업을 진행한다.

### 작업 범위 준수

- 사용자가 명시적으로 요청한 작업 범위를 초과해서는 안 된다.
- 범위 확장이 필요하다고 판단되면, 대안과 근거를 분석/비교하여 사용자에게 보고하고 승인을 얻은 후 진행한다.
- "간단한 개선", "자명한 수정"도 범위 초과의 예외 사유가 될 수 없다.

## 일반 정책

### 경로 작성 규칙

- 모든 경로는 **프로젝트 루트 기준 상대경로**로만 작성해야 한다. 절대경로, `~` 또는 `$HOME` 등 축약 경로 사용을 금지한다.
  - 올바른 예: `backend/src/main/...`, `frontend/src/...`
  - 잘못된 예: `~/project/backend/...`, `/home/user/backend/...`

### 코드 컨벤션

- 반드시 기존 코드에서 사용된 패턴, 네이밍 컨벤션, 들여쓰기 방식, 주석 스타일을 그대로 따라야 한다.
- 새로운 파일을 생성할 때는 같은 도메인에 속한 기존 파일을 템플릿으로 삼아 구조와 스타일을 복제해야 한다.
- 절대 개인적 선호나 임의적 판단으로 기존 코드베이스와 다른 스타일을 도입해서는 안 된다.

### 기술스택별 세부 컨벤션

기술스택별 세부 컨벤션은 아래 문서에 명시되어 있으며, 이를 반드시 준수해야 한다:

- 백엔드: [backend.md](agent/project/backend.md)
- 프론트엔드: [frontend.md](agent/project/frontend.md)

### 도메인 경계 준수

- 작업 대상이 아닌 도메인의 파일은 절대 수정해서는 안 된다. 공통 타입이나 유틸리티가 필요하면 해당 도메인에 신규 파일을 생성하거나, 범용 계층(`global/`, `common/`)에 추가한다.
- 타 도메인의 API, Service, Store를 import만 하고 수정하지 않는 것은 허용한다.

## 작업 후 필수 정책

### 작업 히스토리 기록

- 작업 완료 후, 사용자가 요청한 작업 단위로 히스토리 로그를 기록한다. 상세 규칙과 템플릿은 [history-logging.md](agent/history-logging.md)을 엄격히 준수한다.
- 아무리 간단한 작업이라도 이 절차를 건너뛰는 것은 가드레일 위반으로 간주한다.

### 커밋

- 작업은 논리적 컴포넌트 단위로 분할하여 커밋한다. 상세 규칙은 [commit-convention.md](agent/commit-convention.md)을 엄격히 준수한다.
- 커밋하지 않은 작업은 완료로 간주하지 않는다.
